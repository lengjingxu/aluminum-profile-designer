# 🛠️ 前端技术栈说明

> ⚠️ **本项目为纯前端应用，不依赖任何后端服务。**

---

## 核心原则

**无后端架构** — 所有业务逻辑、数据计算、渲染均在微信小程序客户端完成。

- ✅ 无服务器部署成本
- ✅ 无网络延迟影响计算
- ✅ 数据隐私（不传输至服务器）
- ✅ 离线可用（核心功能不依赖网络）

---

## 技术栈清单

| 技术 | 版本建议 | 用途 | 备注 |
|------|---------|------|------|
| **微信小程序** | 基础库 ≥ 2.25.0 | 主载体 | 使用最新稳定版基础库 |
| **Three.js** | r150+ | 3D渲染引擎 | 需使用小程序适配版（threejs-miniprogram） |
| **Canvas 2D API** | 小程序Canvas组件 | 2D绘图 | 使用新版Canvas 2D接口（Type='2d'） |
| **wx.setStorageSync** | 小程序原生API | 本地数据持久化 | 单条数据上限1MB |
| **jsPDF** | 2.x | PDF生成（前端） | 仅在导出分包中加载 |
| **Vite** | 4.x | 本地开发预览 | 辅助H5预览，不参与小程序构建 |

## 关键技术选型说明

### Three.js 在微信小程序中的使用

Three.js 官方不支持微信小程序环境，需要使用适配层：

- **推荐方案**：[threejs-miniprogram](https://github.com/wechat-miniprogram/threejs-miniprogram)（微信官方适配）
- **注意事项**：
  - 小程序WebGL有性能限制，需控制面数
  - 3D功能放入独立分包，减少主包体积
  - 使用简化的材质和光照模型

### Canvas 2D 绘图

- 使用微信小程序新版 Canvas 2D 接口（`<canvas type="2d">`）
- 分层渲染策略：
  - **背景层**：网格线、画布边框
  - **图元层**：用户绘制的内容
  - **交互层**：吸附线、选中高亮、拖拽辅助

### 本地数据存储

- 使用 `wx.setStorageSync` / `wx.getStorageSync` 进行数据持久化
- 存储内容：
  - 设计方案数据（JSON）
  - 用户设置（配件单价、画布参数）
  - 自定义模板
- 数据压缩：大型设计方案使用压缩存储
- 存储限制：单条1MB，总计10MB，需做好容量管理

### 小程序云开发（可选，仅限文件存储）

> ⚠️ 云开发仅用于分享场景的临时文件存储，**不处理任何业务逻辑**。

- 用途：分享设计方案时，临时存储压缩JSON到云存储
- 不用于：用户认证、数据计算、业务逻辑
- 可选性：核心功能不依赖云开发，仅在分享时可选启用

## 不使用的后端技术

以下技术**明确不使用**：

| 技术 | 原因 |
|------|------|
| Node.js / Express | 纯前端，无需服务端 |
| MySQL / MongoDB | 数据本地存储，无需数据库 |
| Redis | 无服务端缓存需求 |
| Docker / K8s | 无服务端部署需求 |
| REST API / GraphQL | 无后端接口 |
| 用户认证系统 | 无需登录/注册 |

## 构建与部署

```
开发环境：
  微信开发者工具 → 直接构建预览

生产部署：
  微信开发者工具 → 上传代码 → 微信平台审核 → 发布

本地预览（H5模式）：
  Vite → 启动开发服务器 → 浏览器预览（仅用于快速调试2D功能）
```

## 性能约束

| 约束 | 限制值 | 原因 |
|------|--------|------|
| 主包体积 | < 2MB | 微信小程序主包限制 |
| 3D分包体积 | < 1MB | 控制加载时间 |
| 单条本地存储 | < 1MB | wx.setStorageSync限制 |
| 总本地存储 | < 10MB | 微信小程序存储上限 |
| Canvas帧率 | 目标30fps+ | 小程序Canvas性能限制 |
| WebGL面数 | < 50000 | 小程序WebGL性能限制 |
