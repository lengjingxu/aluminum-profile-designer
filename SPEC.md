# ⚙️ 铝型材结构设计器 - 项目规格说明书

> 📐 铝型材结构设计与材料计算工具 · 微信小程序
> 支持3D可视化与配件清单生成

---

## 📋 1. 项目概述

### ✅ 1.1 项目背景

铝型材（工业铝型材）因其轻质、高强度、模块化组装的特性，广泛应用于：

- 🏠 定制家具：衣柜框架、书桌支架、鞋柜结构
- 🖥️ 办公场景：办公桌、显示器支架、线缆管理架
- 🏪 商业展示：展示架、货架、灯箱框架
- 🔧 工业应用：工作台、设备支架、安全围栏

常见铝型材规格包括 **2020、2040、3030、3060、4040、4080、6060、8080** 等，采用标准槽宽设计，配合角码、螺栓等配件实现快速拼装。

### ✅ 1.2 核心价值

本项目的核心价值定位：

| 核心价值 | 说明 |
|---------|------|
| 🔲 所见即所得 | 2D绘图实时映射到3D模型，设计过程直观高效 |
| 🔲 实时3D预览 | 支持旋转/缩放/平移，全方位审视设计成果 |
| 🔲 自动材料计算 | 自动汇总型材长度、配件数量，告别手工算量 |
| 🔲 成本透明 | 实时估算材料+配件+加工费，报价一目了然 |
| 🔲 模板赋能 | 预设常用结构模板，新手也能快速上手 |

## 📊 3. 材料计算模块

### ✅ 3.1 型材长度计算

计算流程：

1. 🔲 读取2D图纸中所有线段长度
2. 🔲 按画布比例换算为实际mm长度
3. 🔲 按铝型材规格分组统计
4. 🔲 汇总同规格型材的总长度（m）
5. 🔲 去重统计：相同长度+规格的型材数量

输出格式示例：

```
规格    | 单根长度(mm) | 数量 | 总长度(m) | 总重量(kg)
2020    | 500          | 8    | 4.0       | 1.20
3030    | 1000         | 4    | 4.0       | 2.92
4040    | 1200         | 2    | 2.4       | 3.10
```

### ✅ 3.2 配件清单计算

支持以下连接方式，每种配件有数量计算逻辑：

| 配件类型 | 说明 | 计算规则 | 默认规格 |
|---------|------|---------|---------|
| 🔲 角码连接件 | 90度角固定 | 两线段夹角≈90°时计数+1 | 匹配型材规格 |
| 🔲 T型连接件 | 三通连接 | 三线段交于一点时计数+1 | 匹配型材规格 |
| 🔲 螺丝/螺栓 | 连接固定 | 每连接点2颗 | M5（默认） |
| 🔲 端盖/堵头 | 型材端面装饰 | 每根型材2个 | 匹配型材规格 |
| 🔲 脚轮/脚杯 | 支撑件 | 统计底部支撑点数量 | M6螺纹 |
| 🔲 面板夹 | 固定层板 | 每面面板4个 | 4040适配 |
| 🔲 弹性扣 | 快速连接 | 每隔300mm一颗，按型材长度÷300计算 | 匹配槽宽 |

计算逻辑伪代码：

```javascript
function calculateAccessories(elements) {
  const result = {};
  
  // 角码：检测90度角连接
  result.angleBrackets = detect90DegreeJoints(elements);
  
  // T型连接件：检测三线交汇点
  result.tConnectors = detectTripleIntersections(elements);
  
  // 螺丝：每个连接点2颗
  result.screws = (result.angleBrackets + result.tConnectors) * 2;
  
  // 端盖：每根型材2个
  result.endCaps = elements.length * 2;
  
  // 脚轮/脚杯：底部支撑点
  result.feet = detectBottomSupportPoints(elements);
  
  // 面板夹：每面板4个
  result.panelClamps = detectPanels(elements) * 4;
  
  // 弹性扣：型材长度÷300
  result.springClips = calculateSpringClips(elements);
  
  return result;
}
```

### ✅ 3.3 成本估算

成本计算公式：

```
总成本 = 型材成本 + 配件成本 + 加工费

型材成本 = Σ(各规格总长度(m) × 单价(元/m))
配件成本 = Σ(各配件数量 × 单价(元/个))
加工费 = (型材成本 + 配件成本) × 加工费比例（默认10%，可调）
```

功能要求：

- 🔲 用户可编辑各配件单价（元/个）和型材单价（元/m）
- 🔲 实时汇总显示：型材成本 + 配件成本 + 加工费
- 🔲 成本明细表：分类显示各项费用
- 🔲 导出报价单为图片（Canvas绘制 → 保存到相册）
- 🔲 导出报价单为PDF（前端jsPDF生成）

## ⚠️ 10. 潜在挑战与风险

### 🔲 技术挑战

| 挑战 | 严重程度 | 缓解方案 |
|------|---------|---------|
| 微信小程序中Three.js性能调优 | 🔴 高 | 使用轻量Three.js适配层、面数限制、LOD策略 |
| 2D到3D自动映射算法 | 🟡 中 | 先实现单视图映射，逐步支持多视图联动 |
| 复杂连接件智能检测 | 🟡 中 | 基于几何关系检测（角度+距离+交点），逐步优化 |
| 微信分包加载策略 | 🟡 中 | 严格控制包体积，Three.js放入独立分包 |

### 🔲 业务挑战

| 挑场 | 说明 | 缓解方案 |
|------|------|---------|
| 配件计算准确性 | 不同厂家的配件规格有差异 | 提供参数可编辑，默认值标注"仅供参考" |
| 型材规格数据维护 | 新规格持续推出 | JSON配置外部化，支持用户自定义添加 |
| 用户学习成本 | DIY用户可能不熟悉工程视图 | 预设模板降低门槛，提供引导教程 |

### 🔲 小程序平台限制

| 限制 | 影响 | 缓解方案 |
|------|------|---------|
| Canvas性能 | 2D绘图帧率受限 | 分层渲染+离屏Canvas |
| 包体积限制 | 主包2MB，总包20MB | 分包策略+资源压缩 |
| Three.js兼容性 | 小程序WebGL限制 | 使用threejs-miniprogram适配库 |
| 分享能力 | 分享链接参数长度受限 | 压缩JSON + 云存储临时文件 |
